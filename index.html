Paso a paso

Como crear un repositorio desde consola de git

Paso 01

Se inicializa la consola de Git
Paso 02

Con cd d: ingresamos a la unidad d
Paso 03

Con mkdir creamos la carpeta tallerGit
Paso 04

Con dir leemos el contenido de la unidad
Paso 05

Con cd tallerGit ingresamos a la carpeta que creamos “tallerGit”
Paso 06

Con git init creamos un nuevo repositorio local vacío
Paso 07

Accedemos a nuestra cuenta de github.com desde un navegador web, con nuestras credenciales
Paso 08

Creamos un repositorio publico en github.com utilizando la interfaz web de github en el navegador
Paso 09

Lo nombramos “Programa-git”

Paso 10

Copiamos la dirección url del repositorio que creamos
Paso 11

Agregamos el repositorio remoto a nuestra carpeta, al repositorio local con git remote add origin + la url que copiamos del repositorio remot
Paso 12

Con git remote -v vemos los repositorios que existen, a pesar de visualizar dos solo hay uno pues están las acciones de fetch y push
Paso 13

Agregamos cambios realizados con git add .
Paso 14

Con git commit -m realizamos un commit
Paso 15

Creamos un nuevo archivo llamado kotlin.txt echo “Nombre del usuario: Gustavo Benitez\nCorreo electrónico: gusadso@gmail.com” >kotlin.txt

Paso 16

No recibir ningún mensaje de error se entiende que se creo el archivo satisfactoriamente, para verificar ingresamos dir y vemos que el archivo efectivamente esta
Paso 17

Para visualizar el contenido del archivo ingresamos cat y el nombre del archivo
Paso 18

Ahora vamos al explorador de archivos y verificamos la creación del archivo en la ruta
Paso 19

Verificamos desde el explorador de archivos de windows
Paso 20

Con git add kotlin.txt agregamos ese archivo al repositorio local; con el comando git add . agregamos todos los archivos que se encuentren.
Paso 21

Realizamos commit con el mensaje “añadido datos” usando el comando git commit -m
Paso 22

Con el comando git push origin master hacemos push al repositorio master
Paso 23

Verificamos en nuestro repositorio remoto que nuestro archivo haya subido de manera satisfactoria.
Paso 24

Verificamos en nuestro repositorio remoto que se encuentre el archivo publicado. Fin del proceso.
